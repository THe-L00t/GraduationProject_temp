  핵심 설계 포인트

  1. 렌더링 파이프라인 (중점과제 반영)

  기획의 Multi-pass 구조를 그대로 반영합니다:

  Frame Start
    ├─ PathTracingPass  →  RadianceBuffer (HDR, RGBA16F)
    │     └─ SampleDistributor: center=16, mid=4, edge=1
    ├─ PostProcessPass  →  Vignette + Blur + Film Grain
    ├─ ToneMappingPass  →  HDR → SDR
    └─ Present (SwapChain)

  PathTracing/ 폴더를 별도로 분리한 이유는 이것이 졸업 프로젝트의 연구 핵심이기 때문입니다. 샘플 분포 전략, Blue Noise,
  노이즈 제어 로직이 독립적으로 발전할 수 있도록 격리합니다.

  2. 멀티쓰레딩 (멀쓰+엔진 반영)

  기획서의 현대 엔진 구조를 따릅니다:

  Main Thread          │ Dedicated Threads      │ Worker Pool (JobSystem)
  ─────────────────────│────────────────────────│─────────────────────────
  Input                │ AudioManager           │ Physics Jobs
  Game Logic           │ ResourceManager        │ Culling Jobs
  Frame Orchestration  │   (비동기 로드)         │ Render Command Jobs
  Scene Graph Update   │                        │ Animation Jobs

  3. 씬 전환 (temp.md 반영)

  SceneManager가 실외→실내→시네마틱 흐름을 관리합니다:

  SceneOutdoor (이끼, 콘크리트, 식물)
      ↓ transition
  SceneIndoor (녹슨 철문, 형광등, 먼지 유리)
      ↓ transition
  SceneCinematic (PBR 효과 극대화 연출)

  실외↔실내 전환 시 머티리얼의 Roughness/Metallic 차이가 PBR 효과를 극대화합니다.

  4. 스프라이트 + 3D 혼합

  SpritePass를 별도 렌더 패스로 분리하여, 3D PBR 환경 위에 2D 스프라이트 몹을 빌보드 방식으로 렌더링합니다. 아이의 상상
  속 이야기라는 세계관과 자연스럽게 어울립니다.
