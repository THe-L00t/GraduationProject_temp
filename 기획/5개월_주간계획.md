# 5개월 엔진 개발 주간 계획 (게임 로직 제외)

> 하루 2~3시간 기준 (주당 약 14~18시간)
> 게임 로직(Player, Monster, NPC, Scene전환 등) 제외 — 엔진/렌더링/인프라에 집중

---

## Phase 1: DX12 기반 세팅 (1~4주)

### 1주차 — Win32 윈도우 + DX12 디바이스 초기화

**작업 목표**
- `Core/Window.h/cpp` — Win32 윈도우 생성, 메시지 루프
- `Core/Application.h/cpp` — 앱 프레임워크 (Init → Update → Render → Destroy)
- `Graphics/Device.h/cpp` — IDXGIFactory, ID3D12Device 생성, Debug Layer 활성화

**완료 기준**: 파란 화면(ClearRenderTargetView)이 뜨는 윈도우

**선행 학습**
- Win32 API 기초 (CreateWindowEx, WndProc 메시지 루프)
- COM 객체와 ComPtr 사용법
- DXGI란 무엇인가 (Factory, Adapter 열거)
- D3D12 Debug Layer / GPU Validation 개념

---

### 2주차 — Command Queue + SwapChain + 펜스 동기화

**작업 목표**
- `Graphics/CommandQueue.h/cpp` — Command Queue, Allocator, List 생성 및 관리
- `Graphics/SwapChain.h/cpp` — 더블/트리플 버퍼링 스왑체인
- GPU-CPU 동기화 펜스(Fence) 구현

**완료 기준**: 매 프레임 RTV를 교체하며 색상이 변하는 화면

**선행 학습**
- DX12 커맨드 모델 (Queue → Allocator → List → Execute → Signal)
- 펜스(Fence) 기반 GPU-CPU 동기화 원리
- 더블/트리플 버퍼링 개념
- DX12 리소스 배리어(Resource Barrier) / 리소스 상태 전이

---

### 3주차 — Descriptor Heap + Root Signature + PSO

**작업 목표**
- `Graphics/DescriptorHeap.h/cpp` — CBV/SRV/UAV/RTV/DSV 힙 관리자
- `Graphics/RootSignature.h/cpp` — Root Signature 빌더
- `Graphics/PipelineState.h/cpp` — Graphics PSO 생성
- 기본 HLSL 버텍스/픽셀 셰이더 작성

**완료 기준**: 화면에 색상이 칠해진 삼각형 렌더링

**선행 학습**
- DX12 Descriptor 시스템 (CBV, SRV, UAV, RTV, DSV 차이)
- Root Signature 개념 (Root Parameter, Descriptor Table, Static Sampler)
- PSO(Pipeline State Object)가 DX11의 무엇을 대체하는지
- HLSL 셰이더 기초 (시맨틱, 레지스터 바인딩)

---

### 4주차 — GPU Buffer + 타이머 + 기본 입력

**작업 목표**
- `Graphics/GPUBuffer.h/cpp` — Vertex/Index/Constant Buffer 래핑 (Upload Heap + Default Heap)
- `Core/Timer.h/cpp` — QueryPerformanceCounter 기반 delta time
- `Input/InputManager.h/cpp` — 키보드/마우스 상태 폴링
- 정육면체(Cube) 렌더링 + 카메라 이동

**완료 기준**: 키보드로 카메라를 움직여 회전하는 큐브를 관찰

**선행 학습**
- DX12 힙 종류 (Upload Heap vs Default Heap vs Readback Heap)
- Constant Buffer 256바이트 정렬 규칙
- 뷰 행렬, 프로젝션 행렬 (DirectXMath)
- Win32 Raw Input 또는 GetAsyncKeyState

- 오디오 스레드 분리 패턴
- 커맨드 리스트 병합과 ExecuteCommandLists
