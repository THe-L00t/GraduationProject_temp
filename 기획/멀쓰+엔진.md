- 커맨드 큐 작성.
- 충돌처리? 
- 사운드 재생
- 게임 시작 시 리소스 로드 
- 입력처리 

### 그럼 이 중에서 동시에 이루어지는건? 
- 입력처리 & 커맨드 큐 작성 & 사운드 & 충돌처리

## GPT 인용
### 1️⃣ 시스템 레벨 멀티쓰레딩 (Subsystem Thread)

이건 독립적으로 돌아가는 시스템이다.
다른 시스템과 의존성이 거의 없다.

보통 별도 스레드를 가진다.

대표적인 것
```
Audio
Resource Streaming
Networking
File IO
Asset Loading
```
예
```
Main Thread
 ├ Game Logic
 ├ Render Setup

Audio Thread
Resource Thread
Network Thread
```
왜 분리하냐면
```
디스크 IO
네트워크 대기
```
같은 blocking 작업이 있기 때문이다.

### 2️⃣ Job System (Worker Thread Pool)

현대 엔진에서 가장 중요한 멀티쓰레딩이다.

구조
```
Worker Thread Pool
```
예
```
8 core CPU

worker 1
worker 2
worker 3
...
worker 8
```
이 스레드들이 job queue에서 작업을 가져간다.

이 시스템이 처리하는 것
```
Physics
Animation
AI
Visibility Culling
Particle Simulation
Skinning
Light Culling
```
즉
```
scene object 기반 계산
```
이 대부분 여기서 처리된다.

### 3️⃣ 렌더링 멀티쓰레딩

이건 GPU 명령 생성 병렬화다.

특히

DirectX12

Vulkan

에서 중요하다.

병렬화 대상
```
Render Command List 생성
```
예
```
thread 1 → shadow pass command
thread 2 → geometry pass command
thread 3 → lighting pass command
thread 4 → post process command
```
그 다음
```
main thread → command queue submit
```
### 실제 엔진 구조 예시

대부분 엔진 구조는 이렇게 생긴다.
```
Main Thread
 ├ Input
 ├ Game Logic
 ├ Frame orchestration

Worker Threads
 ├ Physics Jobs
 ├ Animation Jobs
 ├ AI Jobs
 ├ Culling Jobs
 └ Render Jobs

Dedicated Threads
 ├ Audio
 ├ Resource Streaming
 └ Network
```
프레임 기준으로 보면

보통 CPU 프레임은 이렇게 진행된다.
```
Input

Game Logic

Job System
 ├ Physics
 ├ Animation
 ├ AI
 ├ Culling

Render Command Build

GPU Execute
```
### 멀티쓰레딩을 거의 안하는 부분

다음은 보통 메인 스레드에서 처리된다.
```
Input 처리
Game State 업데이트
Scene Graph 업데이트
Frame orchestration
```
이유
```
데이터 의존성이 강함
현실적인 엔진 CPU 사용 비율
```
대략적인 예 (AAA 게임)
```
Physics          20%
Animation        15%
AI               10%
Culling          10%
Render commands  20%
Audio             5%
Streaming         5%
Game logic       15%
```
여기서
```
Physics
Animation
Culling
Render command
```
이 멀티쓰레딩 핵심 영역이다.

### 정리

일반적인 게임 프레임워크에서 멀티쓰레딩 하는 영역

독립 스레드
```
Audio
Networking
Resource streaming
File IO
```
Job system
```
Physics
Animation
AI
Culling
Particle
Skinning
```
렌더링
```
Render command list 생성
```
마지막으로 하나만 말할게.

많은 사람들이 처음 엔진 만들 때 이렇게 생각한다.
```
physics thread
render thread
logic thread
```
하지만 현대 엔진은 거의 이렇게 안 만든다.

대신
```
job system + worker thread pool
```
구조를 쓴다.
