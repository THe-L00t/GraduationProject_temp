# Path Tracing 기반 의도적 노이즈 분포 및 후처리 효과 연구 아이디어

## 1. 연구 배경

실시간 렌더링에서 **Path Tracing**은 현실적인 광원 계산을 제공하지만 다음과 같은 근본적인 문제가 존재한다.

- 샘플 수가 적으면 **노이즈(noise)** 발생
- 노이즈를 줄이기 위해 샘플 수 증가 필요
- 샘플 수 증가 시 **렌더링 시간 증가**
- 실시간 렌더링 환경에서는 샘플 수 증가에 한계 존재

즉 다음과 같은 트레이드오프가 발생한다.

```
샘플 수 증가 → 노이즈 감소 → 렌더링 시간 증가
샘플 수 감소 → 렌더링 속도 증가 → 노이즈 증가
```

현재 연구 및 산업에서는 이를 해결하기 위해 다음 방법들이 사용된다.

- Temporal accumulation
- Spatial reuse
- Denoising
- Importance sampling
- Sample reuse

하지만 본 아이디어에서는 **노이즈를 완전히 제거하는 대신 "의도적으로 활용"하는 방향**을 탐색한다.

---

# 2. 핵심 아이디어

화면의 위치에 따라 **Path Tracing 샘플 수를 의도적으로 다르게 배분**한다.

예시:

```
화면 중심 → 높은 샘플 수
화면 중간 → 중간 샘플 수
화면 가장자리 → 낮은 샘플 수
```

예시 샘플 분포:

```
center samples = 16
mid samples = 4
edge samples = 1
```

이 경우 결과 화면 특성은 다음과 같다.

```
화면 중심 → 노이즈 거의 없음
화면 주변 → 노이즈 증가
```

이 노이즈를 **연출 효과**로 활용할 수 있다.

예시:

- 꿈 장면
- 기억 장면
- 환각 효과
- 초현실적인 연출

---

# 3. 렌더링 파이프라인 구조

전체 렌더링 구조는 **Multi-pass Rendering** 방식으로 구성한다.

```
Pass 1 : Path Tracing
Pass 2 : Post Processing
Pass 3 : Tone Mapping
Pass 4 : Present
```

구체적인 구조:

```
Ray Generation Shader
↓
Radiance Buffer (HDR Texture)

Denoising / Accumulation
↓
Post Process (Vignette / Blur / Grain)

Tone Mapping
↓
Final Image
```

---

# 4. Pass 1 : Path Tracing

Path Tracing은 **scene lighting 계산**을 담당한다.

출력 버퍼:

```
RWTexture2D<float4> RadianceBuffer
```

Pseudo code:

```
color = PathTrace(ray)

RadianceBuffer[pixel] = float4(color, 1)
```

이 결과는 **HDR 텍스처**로 저장된다.

권장 포맷:

```
RGBA16F
RGBA32F
```

---

# 5. Sample Distribution 전략

픽셀 위치 기반 샘플 분포 예시:

```
distance = distance(pixel, screen_center)

samples = lerp(max_samples, min_samples, distance)
```

예시:

```
center = 16 samples
mid = 4 samples
edge = 1 sample
```

이 방법을 통해 **의도적인 노이즈 분포**를 생성할 수 있다.

---

# 6. Pass 2 : Post Processing

Path Tracing 결과를 기반으로 **screen space 효과**를 적용한다.

대표적인 효과:

- Vignette
- Blur
- Film grain

---

# 7. Vignette 효과

Vignette는 화면 중심 기준 거리로 계산한다.

Pseudo code:

```
dist = distance(uv, center)

vignette = smoothstep(inner_radius, outer_radius, dist)

color *= (1 - vignette)
```

결과:

```
center → 밝음
edge → 어두움
```

---

# 8. Blur 효과

Blur는 여러 방법으로 구현 가능하다.

대표적인 방법:

- Gaussian blur
- Kawase blur
- Bilateral blur

예시 구현:

```
color = 0

for offset in kernel:
color += sample(texture, uv + offset)

color /= kernel_size
```

Blur는 **edge 영역에서만 적용**하는 것도 가능하다.

---

# 9. 노이즈 패턴 제어

단순 랜덤 노이즈 대신 **Blue Noise Sampling**을 사용하면 시각적으로 더 자연스럽다.

특징:

```
low frequency noise 감소
grain-like appearance
```

이는 필름 그레인과 유사한 효과를 만든다.

---

# 10. 확장 가능 아이디어

### 1. Adaptive Sampling

장면 복잡도 기반 샘플 배분

```
complex geometry → sample 증가
simple area → sample 감소
```

---

### 2. Motion-based Sampling

움직임이 큰 영역은 샘플 감소

```
fast motion area → sample 감소
static area → sample 증가
```

---

### 3. Perceptual Rendering

사용자의 시각 인지 기반 샘플 분배

예시:

```
player aim area → sample 증가
peripheral vision → sample 감소
```

---

# 12. 결론

Path Tracing의 노이즈는 일반적으로 제거해야 할 문제로 인식되지만,  
본 아이디어는 이를 **화면 연출 요소로 활용하는 접근**을 제안한다.

핵심 전략은 다음과 같다.

```
공간 기반 샘플 분포 제어

Post-process 효과 결합

Blue noise 기반 노이즈 패턴 제어
```

이를 통해 **성능과 시각적 연출을 동시에 확보**할 가능성이 있다.
